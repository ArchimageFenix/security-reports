Informe Final de Seguridad

Análisis detallado del comportamiento observado en los logs del módulo de autenticación del panel web de AdGuard Home, incluyendo correlación temporal, interpretación semántica, evaluación de riesgo y decisión operativa final.

Resumen Ejecutivo RIESGO BAJO

El análisis de los registros muestra actividad automatizada esporádica, no persistente y sin adaptación. No se identifican indicios de ataque dirigido, fuerza bruta ni enumeración efectiva. El sistema responde de forma determinista y controlada, manteniendo un estado estable.

Ventana de Tiempo Representativa

t0

Primer intento de autenticación fallido (credencial inválida).

t0 + ~60 ms

Segundo intento fallido. Ritmo no humano.

t0 + ~120 ms

Activación inmediata del cooldown (60 minutos).

t0 + 200–900 ms

Ráfaga de intentos bloqueados. El cliente no detecta el bloqueo.

t0 + ~1 s

Fin del contacto. No se observan más intentos.

Análisis Indirecto del Origen Más Ruidoso

  • Comportamiento completamente automatizado.
  • Proceso sin estado ni lógica de adaptación.
  • La ráfaga corresponde a un único job de escaneo.
  • El abandono ocurre por finalización del proceso, no por aprendizaje.

Correlación Temporal Global

  • Eventos separados por días, no por horas.
  • Frecuencia extremadamente baja.
  • Ausencia total de persistencia o seguimiento.
  • Silencio prolongado (>29 h) tras el último evento.

Este patrón es consistente con escaneo de fondo distribuido y descarta campañas, targeting o presión sostenida sobre el servicio.

Evaluación de Riesgo

  • Compromiso: Muy bajo
  • Fuerza bruta: Nula
  • Enumeración: Muy baja
  • Impacto operativo: Despreciable
  • Riesgo residual: Estructural (0-day)

Decisión Operativa

El panel web permanece expuesto por necesidad funcional. Dadas las condiciones observadas (baja frecuencia, ausencia de presión y respuesta correcta del sistema), esta decisión es técnicamente defendible y no constituye negligencia.

La opción de ocultar o aislar el panel queda disponible como mejora futura, no como acción urgente.

Conclusión técnica final

El sistema se encuentra en un estado de equilibrio estable. La actividad observada corresponde a ruido automatizado común de Internet, sin señales de interés sostenido ni capacidad ofensiva. La arquitectura actual mitiga eficazmente este ruido con herramientas ligeras y un modelo coherente.

No se recomiendan cambios inmediatos. La observación continua y el ajuste incremental siguen siendo la estrategia correcta.
